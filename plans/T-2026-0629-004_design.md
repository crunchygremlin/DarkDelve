# T-2026-0629-004 Design: LLM Model GPU Persistence

### 4.2 `OllamaService.generate` (src/infrastructure/external/ollama_service.py, line 78)

```python
def generate(self, prompt: str, **kwargs) -> str:
    """Generate text using Ollama."""
    try:
        payload = {
            "model": self.model,
            "prompt": prompt,
            "stream": False,
            "temperature": kwargs.get("temperature", 0.7),
            "top_p": kwargs.get("top_p", 0.9),
            "num_predict": kwargs.get("num_predict", 1024),
        }
        # Add keep_alive if provided (backward compatible: older Ollama ignores unknown keys)
        keep_alive = kwargs.get("keep_alive", self.keep_alive)
        if keep_alive is not None:
            payload["keep_alive"] = keep_alive

        resp = requests.post(
            f"{self.base_url}/api/generate",
            json=payload,
            timeout=kwargs.get("timeout", 30)
        )
        if resp.status_code == 200:
            self._model_loaded = True
            return resp.json().get('response', '').strip()
    except Exception as e:
        raise ExternalServiceException(f"Generation error: {e}")
    return ""
```

### 4.3 `OllamaService.purge` (src/infrastructure/external/ollama_service.py, NEW â€” insert before `stop()`)

```python
def purge(self) -> bool:
    """Unload model from GPU memory by sending keep_alive=0.
    
    Returns True if purge succeeded, False otherwise.
    Does not raise â€” errors are logged and swallowed so cleanup
    never fails.
    """
    if not self._started or not self._model_loaded:
        return True
    try:
        payload = {
            "model": self.model,
            "prompt": "",
            "stream": False,
            "keep_alive": "0",
        }
        resp = requests.post(
            f"{self.base_url}/api/generate",
            json=payload,
            timeout=10
        )
        if resp.status_code == 200:
            self._model_loaded = False
            return True
    except Exception as e:
        # Swallow errors during cleanup â€” never crash on exit
        pass
    return False
```

### 4.4 `OllamaService.stop` (src/infrastructure/external/ollama_service.py, line 114)

```python
def stop(self) -> None:
    """Stop the Ollama server."""
    # Purge model from GPU before terminating server
    self.purge()
    if self._process:
        self._process.terminate()
        self._process.wait(timeout=5)
        self._started = False
```

### 7.5 Backward Compatibility

- Older Ollama versions that don't support `keep_alive` will ignore the unknown key in the JSON payload (Ollama's API ignores unrecognized fields). No error will occur.
- If `keep_alive` is set to `None`, the key is omitted from the payload entirely, restoring pre-feature behavior.
- All existing callers of `generate()` and `generate_json()` use `**kwargs` and never pass `keep_alive`, so they inherit the instance default. No caller changes needed.

## 8. Risks and Mitigations

| Risk | Mitigation |
|------|------------|
| Older Ollama versions may not support `keep_alive` | Ollama API ignores unknown JSON keys; also, setting `keep_alive=None` omits the key entirely |
| `purge()` could fail if Ollama server is already dead | `purge()` swallows all exceptions and returns `False`; cleanup never crashes |
| `purge()` sends an empty prompt which might log an error server-side | This is the documented Ollama API pattern for unloading models; the empty prompt is intentional |
| `stop()` now takes longer because it calls `purge()` before terminating | `purge()` has a 10-second timeout; if it hangs, `stop()` will still terminate the process afterward |
| Two parallel classes (`OllamaService` and `EmbeddedOllama`) must both be updated | Both classes are modified identically; this is a known tech debt from the ongoing modular refactoring (see architecture/system_overview.md) |
| GPU memory may not be freed if game crashes (no signal handler) | `stop()` calls `purge()` which handles the common case; for crash scenarios, Ollama's own TTL will eventually unload the model after the `keep_alive` duration expires |
| `_model_loaded` flag could become stale if model is unloaded externally | Flag is conservative â€” `purge()` only skips the API call if flag is `False`; worst case is a no-op API call with `keep_alive=0` |
