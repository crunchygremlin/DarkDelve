```python
def _build_dm_evolution_context(self, depth: int) -> Optional[Dict[str, Any]]:
    """
    Build a context dict summarizing previous levels for the DM.

    Returns None if no previous level data exists.
    Otherwise returns a dict with:
    - previous_levels: list of summarized LevelRecord dicts
    - performance_summary: str describing player performance
    - difficulty_adjustment: float multiplier for scaling
    - narrative_continuity: str suggesting thematic direction
    """
    if not self.dm_context or not self.dm_context.get("levels"):
        return None

    levels = self.dm_context["levels"]

    # Summarize last 3 levels (bounded context)
    recent_levels = levels[-3:]

    previous_levels_summary = []
    for lvl in recent_levels:
        summary = {
            "depth": lvl.get("depth", 0),
            "theme": lvl.get("theme_name", "unknown"),
            "monster_theme": lvl.get("monster_theme", "unknown"),
            "monsters_killed": lvl.get("monsters_killed", 0),
            "total_monsters": lvl.get("total_monsters", 0),
            "turns_taken": lvl.get("turns_taken", 0),
            "damage_taken": lvl.get("damage_taken", 0),
            "close_calls": lvl.get("close_calls", 0),
            "difficulty_rating": lvl.get("difficulty_rating", 5.0),
        }
        previous_levels_summary.append(summary)

    # Compute performance on most recent level
    latest = recent_levels[-1] if recent_levels else {}
    performance_summary = self._compute_performance_summary(latest)

    # Compute difficulty adjustment
    difficulty_adjustment = self._compute_difficulty_adjustment(latest)

    # Build narrative continuity suggestion
    narrative_continuity = self._build_narrative_continuity(recent_levels)

    return {
        "previous_levels": previous_levels_summary,
        "performance_summary": performance_summary,
        "difficulty_adjustment": difficulty_adjustment,
        "narrative_continuity": narrative_continuity,
        "target_depth": depth,
    }
```

#### 3.1.6 New method: `Game._compute_performance_summary`

```python
def _compute_performance_summary(self, level_record: Dict[str, Any]) -> str:
    """
    Compute a human-readable performance summary from a LevelRecord.

    Returns a string like:
    - "Player dominated: killed 8/8 monsters, took 5 damage, 0 close calls."
    - "Player struggled: killed 3/8 monsters, took 45 damage, 2 close calls."
    - "No previous level data."
    """
    if not level_record:
        return "No previous level data."

    killed = level_record.get("monsters_killed", 0)
    total = level_record.get("total_monsters", 0)
    damage = level_record.get("damage_taken", 0)
    close_calls = level_record.get("close_calls", 0)
    turns = level_record.get("turns_taken", 0)

    if total == 0:
        ratio = 1.0
    else:
        ratio = killed / total

    if ratio >= 0.8 and damage < 20 and close_calls == 0:
        dominance = "dominated"
    elif ratio >= 0.5 and damage < 40:
        dominance = "handled well"
    elif ratio < 0.3 or damage > 50 or close_calls >= 2:
        dominance = "struggled"
    else:
        dominance = "managed"

    return (
        f"Player {dominance}: killed {killed}/{total} monsters, "
        f"took {damage} damage, {close_calls} close calls, "
        f"completed in {turns} turns."
    )
```

#### 3.1.7 New method: `Game._compute_difficulty_adjustment`

```python
def _compute_difficulty_adjustment(self, level_record: Dict[str, Any]) -> float:
    """
    Compute a difficulty multiplier based on player performance.

    Returns a float:
    - 1.3 if player dominated (increase difficulty more)
    - 1.1 if player handled well (slight increase)
    - 1.0 if player managed (normal scaling)
    - 0.8 if player struggled (decrease difficulty)
    - 1.0 if no data (default)

    The multiplier is clamped to [0.5, 2.0] for safety.
    """
    if not level_record:
        return 1.0

    killed = level_record.get("monsters_killed", 0)
    total = level_record.get("total_monsters", 0)
    damage = level_record.get("damage_taken", 0)
    close_calls = level_record.get("close_calls", 0)

    if total == 0:
        ratio = 1.0
    else:
        ratio = killed / total

    # Base adjustment from kill ratio
    if ratio >= 0.8 and damage < 20 and close_calls == 0:
        adjustment = 1.3
    elif ratio >= 0.5 and damage < 40:
        adjustment = 1.1
    elif ratio < 0.3 or damage > 50 or close_calls >= 2:
        adjustment = 0.8
    else:
        adjustment = 1.0

    # Clamp
    return max(0.5, min(2.0, adjustment))
```

#### 3.1.8 New method: `Game._build_narrative_continuity`

```python
def _build_narrative_continuity(self, recent_levels: List[Dict[str, Any]]) -> str:
    """
    Build a narrative continuity suggestion from recent level themes.

    Returns a string like:
    - "Previous levels: Goblin Warrens (goblins), Crypt of Shadows (undead). Continue the descent into darker themes."
    - "First level — no previous narrative."
    """
    if not recent_levels:
        return "First level — no previous narrative."

    theme_descriptions = []
    for lvl in recent_levels:
        theme_name = lvl.get("theme_name", "unknown")
        monster_theme = lvl.get("monster_theme", "unknown")
        theme_descriptions.append(f"{theme_name} ({monster_theme})")

    themes_str = ", ".join(theme_descriptions)

    # Suggest escalation
    depth = recent_levels[-1].get("depth", 1)
    if depth <= 2:
        direction = "Introduce new threats and deepen the atmosphere."
    elif depth <= 4:
        direction = "Escalate the danger — the dungeon grows more hostile."
    else:
        direction = "The deepest horrors await — push the player to their limits."

    return f"Previous levels: {themes_str}. {direction}"
```

```python
def _record_level_performance(self):
    """
    Record the current level's performance into dm_context['levels'].

    Called when the player descends to a new level (before generating the new level).
    Computes turns taken, damage taken, kills, close calls from the current
    dm_context counters and the player's current state.
    """
    if not hasattr(self, 'dm_context') or self.dm_context is None:
        return
    if not hasattr(self, 'player') or self.player is None:
        return

    start_turn = self.dm_context.get("current_level_start_turn", 0)
    start_hp = self.dm_context.get("current_level_start_hp", self.player.hp)

    turns_taken = max(0, self.turn - start_turn)
    damage_taken = max(0, self.dm_context.get("current_level_damage_taken", 0))

    # If player took damage but start_hp wasn't set properly, estimate
    if damage_taken == 0 and start_hp > self.player.hp:
        damage_taken = start_hp - self.player.hp

    level_record = {
        "depth": getattr(self.state, 'depth', 1),
        "theme_name": getattr(self.current_theme, 'name', 'unknown') if hasattr(self, 'current_theme') and self.current_theme else 'unknown',
        "monster_theme": getattr(self.current_theme, 'monster_theme', 'unknown') if hasattr(self, 'current_theme') and self.current_theme else 'unknown',
        "monsters_killed": self.dm_context.get("current_level_kills", 0),
        "total_monsters": self.dm_context.get("total_level_monsters", 0),
        "turns_taken": turns_taken,
        "damage_taken": damage_taken,
        "close_calls": self.dm_context.get("current_level_close_calls", 0),
        "difficulty_rating": getattr(self.current_theme, 'difficulty', 5.0) if hasattr(self, 'current_theme') and self.current_theme else 5.0,
    }

    self.dm_context["levels"].append(level_record)

    # Bound the history to last 10 levels to prevent unbounded growth
    if len(self.dm_context["levels"]) > 10:
        self.dm_context["levels"] = self.dm_context["levels"][-10:]
```

```python
def design_evolved_level(
    self,
    evolution_context: Dict[str, Any],
    depth: int,
) -> Dict[str, Any]:
    """
    Design a level using evolution context from previous levels.

    Args:
        evolution_context: Dict from Game._build_dm_evolution_context()
        depth: Target depth

    Returns:
        Dict with evolved level config, or {} on failure
    """
    prompt = self.build_evolution_prompt(evolution_context, depth)

    # Estimate tokens and check headroom
    prompt_tokens = self.logger.estimate_prompt_tokens(prompt) if self.logger else 0
    headroom_diag = self.logger.check_headroom(prompt) if self.logger else None

    start = time.time()
    try:
        response = self.ollama.generate(prompt, timeout=30)
        latency = (time.time() - start) * 1000
        response_tokens = estimate_tokens(response) if response else 0

        # Parse JSON response
        json_start = response.find('{')
        json_end = response.rfind('}') + 1
        if json_start >= 0 and json_end > json_start:
            level_config = json.loads(response[json_start:json_end])

            # Log success
            if self.logger:
                self.logger.log_call(LLMCallLog(
                    call_id=f"evolved_level_{depth}_{time.time()}",
                    timestamp=start,
                    context="evolved_level_design",
                    entity_id=None,
                    prompt_summary=prompt[:200],
                    response_summary=response[:200],
                    latency_ms=latency,
                    tokens_used=prompt_tokens + response_tokens,
                    success=True,
                    prompt_tokens=prompt_tokens,
                    response_tokens=response_tokens,
                    context_before_tokens=headroom_diag.max_context_tokens - headroom_diag.headroom_tokens if headroom_diag else 0,
                    context_after_tokens=(headroom_diag.max_context_tokens - headroom_diag.headroom_tokens + response_tokens) if headroom_diag else 0,
                    context_headroom=(headroom_diag.max_context_tokens - (headroom_diag.max_context_tokens - headroom_diag.headroom_tokens + response_tokens)) if headroom_diag else 0,
                    model=self._model_name,
                    temperature=self._temperature,
                ))

            return level_config

        # JSON parse failure
        if self.logger:
            self.logger.log_call(LLMCallLog(
                call_id=f"evolved_level_parse_fail_{time.time()}",
                timestamp=start,
                context="evolved_level_design",
                entity_id=None,
                prompt_summary=prompt[:200],
                response_summary="",
                latency_ms=(time.time() - start) * 1000,
                tokens_used=prompt_tokens,
                success=False,
                error="Failed to parse JSON",
                model=self._model_name,
                temperature=self._temperature,
            ))
        return {}

    except Exception as e:
        latency = (time.time() - start) * 1000
        if self.logger:
            self.logger.log_call(LLMCallLog(
                call_id=f"evolved_level_error_{time.time()}",
                timestamp=start,
                context="evolved_level_design",
                entity_id=None,
                prompt_summary=prompt[:200],
                response_summary="",
                latency_ms=latency,
                tokens_used=prompt_tokens,
                success=False,
                error=str(e),
                model=self._model_name,
                temperature=self._temperature,
            ))
        return {}
```

### 4.1 Difficulty Adaptation Algorithm

```
function compute_difficulty_adjustment(level_record):
    if no level_record: return 1.0
    ratio = killed / total_monsters
    
    if ratio >= 0.8 AND damage < 20 AND close_calls == 0:
        return 1.3    # Player dominated — ramp up
    elif ratio >= 0.5 AND damage < 40:
        return 1.1    # Player handled well — slight increase
    elif ratio < 0.3 OR damage > 50 OR close_calls >= 2:
        return 0.8    # Player struggled — ease off
    else:
        return 1.0    # Balanced — normal scaling
    
    return clamp(result, 0.5, 2.0)
```

### 4.2 Context Bounding

- Only the last 3 levels are included in the evolution prompt
- Only the last 10 levels are stored in `dm_context["levels"]`
- Each level record is a fixed-size dict (~8 keys)
- Total bounded memory: 10 records × ~200 bytes = ~2KB max

## 7. Integration Notes

### 7.1 Integration with `Game.generate_level()`

The `dm_context` snapshot is taken at the start of `generate_level()` so that when the player descends from level N to level N+1, the performance data from level N is captured before the new level overwrites entity state.

### 7.2 Integration with `_generate_standard_level()`

The evolution context is built AFTER the theme is generated but BEFORE the monster roster is finalized. This allows the evolved roster to override the procedural one if the LLM responds in time. Since the LLM call is async (enqueued to `llm_request_queue`), the procedural roster is always available as a fallback.

### 7.3 Integration with `llm_worker_func`

Two new request types are added:
- `evoked_roster`: Synchronous LLM call that returns a full evolved level config
- `behavior_with_context`: Behavior script generation with DM narrative prepended to social context

Both follow the existing pattern: read from `request_queue`, process, put to `response_queue`, log to `llm_logger`.

### 7.4 Integration with `DungeonMasterAgent`

The agent gains three new methods that use its existing `ollama`, `logger`, and `_model_name`/`_temperature` attributes. No new dependencies are introduced.

### 7.5 Backward Compatibility

- All `dm_context` access is guarded by `hasattr(self, 'dm_context')` and `self.dm_enabled` checks
- If DM is disabled, `dm_context` is never populated and procedural generation is unchanged
- If LLM fails, the empty dict `{}` return triggers existing fallback paths
- Existing tests that mock `dm_agent` will not be affected since the new methods are only called when `dm_enabled` is True

## 8. Risks and Mitigations

### 8.1 Risk: Performance tracking hooks in combat code could introduce bugs

**Mitigation**: All tracking code is guarded by, 'dm_context')` and `self.dm_enabled`. The tracking is additive (incrementing counters) and doesn't modify existing combat logic. If `dm_context` is None or DM is disabled, the hooks are no-ops.

### 8.2 Risk: dm_context could grow unboundedly

**Mitigation**: Context is explicitly bounded:
- `dm_context["levels"]` is capped at 10 entries (trimmed in `_record_level_performance`)
- Evolution prompt only includes the last 3 levels
- Each level record is a fixed-size dict with 8 keys

### 8. Risk: LLM timeout could block level generation

**Mitigation**: The evolved roster request is enqueued to `llm_request_queue` (non-blocking). The procedural roster is generated synchronously as a fallback. If the LLM response arrives later, it's ignored (the level is already generated). Future work could apply the evolved config to subsequent levels.

### 8.4 Risk: New request types in llm_worker could conflict with existing ones

**Mitigation**: The new types (`evolved_roster`, `behavior_with_context`) are distinct strings. The `elif` chain in `llm_worker_func` ensures only one handler runs per request. Existing request types (`behavior`, `level_design`, `floor1_description`, `map_generation`) are unchanged.

### 8.5 Risk: Test breakage from new attributes on Game

**Mitigation**: All new attributes are initialized in `__init__` and `initialize()`. Tests that construct `Game()` will have default values. The `dm_context` dict is initialized to a safe empty state.

### 8.6 Risk: Coordinate system confusion in FOV/explored reset

**Mitigation**: The FOV reset logic (line 2599-2602) is unchanged. The `dm_context` snapshot happens before FOV reset, so player position data is captured correctly.

### 8.7 Reference: [architecture/gotchas.md](../architecture/gotchas.md)

- **Import paths**: All imports use `src.` prefix ✓
- **dungeon_map[x, y]**: True=wall, False=floor — not directly relevant here ✓
- **FOV pov=(safe_x, safe_y)**: Not modified ✓
- **Console rendering**: Not modified ✓
- **Non-blocking LLM**: All new LLM calls use the async queue pattern ✓
