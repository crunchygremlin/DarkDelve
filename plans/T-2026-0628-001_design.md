# T-2026-0628-001: Wire DM LLM into Live Game with Observability

## Goal
Connect the existing DungeonMasterAgent and LevelDesignService to the live game loop (`darkdelve.py`) so the LLM actively participates in gameplay, and build a comprehensive observability system that logs every LLM interaction.


**New method `_process_llm_responses()`:**
```python
def _process_llm_responses(self):
    while not self.llm_response_queue.empty():
        try:
            result = self.llm_response_queue.get_nowait()
            entity_id = result['entity_id']
            for entity in self.entities:
                if getattr(entity, 'id', None) == entity_id:
                    if 'behavior_script' in result:
                        entity.behavior_script = result['behavior_script']
                    break
        except Empty:
            break
```

## Fallback Strategy
- If LLM is unavailable (connection error, timeout): fall back to existing hardcoded flee/patrol AI
- If `max_calls_per_turn` exceeded: skip LLM requests for remaining entities that turn
- If `enable_behavior_generation: false`: no behavior LLM calls, entities use hardcoded AI
- If `enable_level_design: false`: level generation stays purely procedural
- If `dm_enabled: false`: entire DM system is off, zero behavior change
