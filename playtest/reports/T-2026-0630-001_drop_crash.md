# T-2026-0630-001 — DropCommand / UseCommand Crash

**Title:** `DropCommand` crashes with `'Entity' object has no attribute 'get_item_count'`

**Severity:** Crash / game-breaking

**Status:** ✅ VERIFIED FIXED — all tests pass, no regressions

---

## Bug Summary

Pressing `D` (drop) or `U` (use) in the inventory screen crashed the game immediately:

```
AttributeError: 'Entity' object has no attribute 'get_item_count'
```

at [`src/application/game_commands/drop_command.py:29`](src/application/game_commands/drop_command.py:29).

The same bug existed in [`use_command.py:28,50,73,100,118,143`](src/application/game_commands/use_command.py).

---

## Root Cause

There are **two** classes named `Entity`/`Player` in the codebase:

| Class | Module | Has `get_item_count`? | Used at runtime as `self.player`? |
|-------|--------|-----------------------|-----------------------------------|
| `Entity` | [`darkdelve.py:763`](darkdelve.py:763) (local `@dataclass`) | **NO** | **YES** — instantiated at [`darkdelve.py:2330`](darkdelve.py:2330) |
| `Player` | [`src/domain/entities/player.py:9`](src/domain/entities/player.py:9) | YES | NO — never instantiated by the live game |

`DropCommand` and `UseCommand` were written against the **domain** `Player` class, but the **runtime** `Entity` lacked the required methods.

Additionally, `DropCommand` accessed `self.player.x`/`self.player.y` directly, but the domain `Player` uses `self.position` (a `Position` object).

---

## Fix Applied

### 1. Runtime `darkdelve.Entity` — 5 new methods ([`darkdelve.py:833-882`](darkdelve.py:833))

- `get_item_count(item)` — counts items in `self.inventory.items` matching item id
- `add_item(item)` — delegates to `self.inventory.add_item()`
- `drop_item(item)` — checks count then delegates to `self.inventory.remove_item()`
- `use_item(item)` — applies heal effects from `item.effect`, removes consumables
- `remove_effect(effect)` — removes from `self.effects` dict

### 2. Runtime `darkdelve.Item` — 4 new properties ([`darkdelve.py:483-510`](darkdelve.py:483))

- `is_droppable` — True unless item is equipped
- `is_usable` — True if `special_effect` is not None
- `consumable` — True for POTION/SCROLL/FOOD types
- `effect` — returns formatted string like `"heal+20"`

### 3. Domain `Item` — `is_droppable` property ([`src/domain/entities/item.py:37`](src/domain/entities/item.py:37))

Added to match the runtime Item interface expected by DropCommand.

### 4. Domain `Player` — `drop_item` method ([`src/domain/entities/player.py:253`](src/domain/entities/player.py:253))

Delegates to `self.inventory.remove_item(item_id)` after checking count.

### 5. `DropCommand` — position-agnostic helper ([`src/application/game_commands/drop_command.py:12-17`](src/application/game_commands/drop_command.py:12))

Added `_get_player_position()` helper that works with both:
- Domain `Player` (uses `.position` attribute)
- Runtime `Entity` (uses `.x`/`.y` attributes)
